## 一、改进概述

本项目在原始 YOLO11 模型的基础上，引入了 **CBAM（Convolutional Block Attention Module）注意力机制**，通过在 backbone 和 head 的关键位置添加注意力模块，提升模型对重要特征区域的关注度，从而改善目标检测的精度。

---

## 二、改进原理

### 2.1 CBAM 模块介绍

CBAM 是一种轻量级的注意力模块，可以即插即用地集成到任何 CNN 架构中。它包含两个串联的子模块：

1. **通道注意力（Channel Attention）**
   - 通过对特征图进行全局平均池化和最大池化
   - 学习每个通道的重要性权重
   - 让模型关注更有判别力的通道特征

2. **空间注意力（Spatial Attention）**
   - 在通道注意力之后，对特征图进行空间维度的加权
   - 学习每个空间位置的重要性权重
   - 让模型关注更重要的空间区域

### 2.2 改进动机

原始 YOLO11 模型虽然性能优秀，但在以下场景仍有提升空间：
- 小目标检测（特征不够显著）
- 复杂背景下的目标检测（容易受背景干扰）
- 遮挡场景（需要更强的特征表达能力）

CBAM 模块可以通过注意力机制，让模型：
- 更关注包含目标的特征通道
- 更关注目标所在的空间区域
- 抑制背景噪声和无关特征

## 三、网络结构修改

### 3.1 Backbone 改进

在 backbone 的 3 个关键位置添加 CBAM 模块：

| 位置 | 模块名称 | 输入特征 | 作用 |
|------|----------|----------|------|
| P3 输出后 | CBAM-1 | 256 通道 | 增强浅层特征，提升小目标检测 |
| P4 输出后 | CBAM-2 | 512 通道 | 增强中层特征，平衡精度与速度 |
| P5 输出后 | CBAM-3 | 1024 通道 | 增强深层语义特征，提升大目标检测 |

### 3.2 Head 改进

在 neck/heaad 的 2 个特征融合位置添加 CBAM 模块：

| 位置 | 模块名称 | 输入特征 | 作用 |
|------|----------|----------|------|
| P3 融合后 | CBAM-4 | 256 通道 | 融合后的浅层特征增强 |
| P4 融合后 | CBAM-5 | 512 通道 | 融合后的中层特征增强 |

### 3.3 网络结构图

```
原始 YOLO11 Backbone:
┌─────────────────────────────────────────┐
│ Conv → Conv → C3k2 → Conv → C3k2 →     │
│ Conv → C3k2 → Conv → C3k2 → SPPF → C2PSA│
└─────────────────────────────────────────┘

改进后 YOLO11-CBAM Backbone:
┌─────────────────────────────────────────────────────┐
│ Conv → Conv → C3k2 → [CBAM] → Conv → C3k2 → [CBAM]→│

└─────────────────────────────────────────────────────┘
                    ↑ 新增的 CBAM 模块
```

**修改文件：`ultralytics/nn/tasks.py`**

这个文件是 YOLO 模型解析的核心文件，负责将 YAML 配置文件解析为实际的 PyTorch 模型。

**修改 1：添加 CBAM 导入（第 21 行）**
```python
from ultralytics.nn.modules import (
    AIFI,
    C1,
    C2,
    C2PSA,

    ELAN1,

```

**修改 2：添加 CBAM 到 base_modules 集合（第 1612 行）**
```python
base_modules = frozenset(
    {
        Classify,
        Conv,
        ...
        A2C2f,
        CBAM,  # ← 新增：将 CBAM 添加到基础模块集合
    }

```

**为什么要修改这里？**
- `parse_model` 函数通过 `globals()[m]` 来获取模块类
- 如果不添加导入，解析 YAML 时会找不到 CBAM 类，导致 `KeyError`
- `base_modules` 集合用于识别哪些模块不需要重复次数处理

### 4.3 CBAM 模块实现

CBAM 模块已经存在于 Ultralytics 代码库中：

**文件位置：** `ultralytics/nn/modules/conv.py` (第 583-613 行)

```python
class CBAM(nn.Module):
    """Convolutional Block Attention Module."""
    
    def __init__(self, c1, kernel_size=7):
        """Initialize CBAM with given parameters.
        
        Args:
            c1 (int): Number of input channels.
            kernel_size (int): Size of the convolutional kernel for spatial attention.
        """
        super().__init__()
        self.channel_attention = ChannelAttention(c1)
        self.spatial_attention = SpatialAttention(kernel_size)

    def forward(self, x):
        """Apply channel and spatial attention sequentially to input tensor."""
        return self.spatial_attention(self.channel_attention(x))
```

### 4.4 模型配置文件

**文件位置：** `ultralytics/cfg/models/11/yolo11_cbam_55230316.yaml`

关键配置示例：
```yaml
backbone:
  - [-1, 1, Conv, [64, 3, 2]]              # 0-P1/2
  - [-1, 1, Conv, [128, 3, 2]]             # 1-P2/4
  - [-1, 2, C3k2, [256, False, 0.25]]      # 2
  - [-1, 1, CBAM, [256]]                   # 3-CBAM: 添加 CBAM 模块
  - [-1, 1, Conv, [256, 3, 2]]             # 4-P3/8
  ...
```

配置格式说明：
- `[from, repeats, module, args]`
- `from`: 输入来源（-1 表示上一层）
- `repeats`: 重复次数（CBAM 为 1）
- `module`: 模块名称（必须与 `tasks.py` 中导入的类名一致）
- `args`: 模块参数列表
